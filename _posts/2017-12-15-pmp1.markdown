# 前因
这些年一直忙忙碌碌，偶尔动过去考证的念头，但基本上也就想想。总是觉得有那时间不如多写写代码，多看看大牛的源码。不是说认证没价值，而是我这些年呆的公司和环境使我觉得认证学到的知识貌似也没有用武之地。（个人成见，随着这次备考的学习，这个成见也许会慢慢松动）。事情的发生要追朔到上上周的周五了，一个研发同事说起打算报考下一年3月份的PMP认证考试，周末要去报名。一刹那间，考证的念头又一次涌现，有人一起学也正好相互督促，毕竟简历上一个证都没有多难看啊（要说这个时候就满脑子自我提升什么的，那太假了，简单的说，原始需求就是简历上多一行字，“通过xxxx年PMP认证”）。好吧，冲动是魔鬼，基本上没有天多考虑，跟同事聊了几句，就请她帮我周末一起报名，周末又沟通确认了一次，交了钱，好吧，就这么在坑里了。项目（把这次备考当成项目的话）发起人，就是我，项目经理也是我，项目成员、项目管理团队一切的一切都只有我。项目的战略目标，在那个时刻就只有两条，A、简历上多一条记录。B、几千大洋啊，不能白花了啊。

# 经过

# 体会
毕竟才过了1个星期，没太多好说的。简单说几句体会吧。
- 刚开始学有点不习惯，因为PMBOK是英文翻译的，虽然句法没问题，但总感觉别扭(听老师说现在翻译的比过去强多了，逻辑上很严谨，我们得适应)。学到第4天的时候，开始觉得有意思了，很多知识点和自己从业这些年的经历能对照起来了，虽然从业经历里碰到的大多数负面案例，但也算知识找到了着力点。另外，系统的去学习项目管理的理论后，能感受到理论的严谨“美”了。我是那种理解促进记忆的人，知识越系统、越严谨，我反而觉得好学。当年考毛泽东思想、邓小平理论时，那个痛苦啊，就是觉得理论的深度无法理解（不多说了）。
- PMBOK是工具书，实际应用时，里面的流程和工具肯定是要裁剪的，如果应用到实际的场景里，估计是书本上学不到的东西了，得多年的实践去体会吧。用书上的话说，项目经理很重要的一项能力就是“个人能力”，这个我理解这个个人能力虽然也是后天修炼而来，但影响的因素太多了，和性格、人生经历甚至价值观、世界观都有关系，没有很强烈的意愿和多年的不断身体力行，这个个人能力恐怕不好提升。不过这个看法也未必是绝对的。今天老师谈到学习项目管理的好处时，说到会增强人的理性思维能力。进而问到理性的反面时什么，大多数同学回答感性。这就是大多数人（也包括我）的误区。理性的反面时不理性，是鲁莽。不论人是否感性，理性只有越多越好。所以我潜意识里，自己是个做技术的，跟代码跟机器打交道时我的爱好，不愿意去做项目经理那种和各部门天天沟通的“虚活”。这种看法希望能随着学习有所改变。起码可以试着用项目管理的思维方式去解决自己的问题（广义上来说，很多事情都可以看作是项目）。  
- 坑很深，不容易爬出来啊。书的厚度也不是问题，我通读了几天，虽然感觉内容很多，也没感觉不可逾越。但今天上课听老师讲了几道往年的题，才体会到套路的深度。对知识的理解必须特别到位，否则是很难出坑的。靠题海是无法解决问题的。出题点太多了，更何况现在题库是不公开的，也没有题库也做。

# 总结
不能多说了，毕竟才这么几天。再次给自己鼓劲，钱已经花了，证和知识缺一不可。睡觉去啦。
